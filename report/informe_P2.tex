\subsection{Definici\'on del problema}
El monitoreo bioac\'ustico pasivo constituye una herramienta clave para la
conservaci\'on de la biodiversidad, al permitir estimar la presencia de especies
en un h\'abitat sin intervenci\'on humana directa. En el presente trabajo se aborda
la identificaci\'on autom\'atica de especies faun\'isticas a partir de registros
eco-ac\'usticos preprocesados.

Formalmente, cada muestra se representa mediante un vector de caracter\'isticas
$\mathbf{x} \in \mathbb{R}^{64}$ compuesto por las variables
$\{\texttt{mel\_0}, \dots, \texttt{mel\_63}\}$, que resumen informaci\'on espectral
en escala Mel. El problema se formula como una tarea de \textbf{clasificaci\'on
supervisada multiclase} sobre la variable objetivo \texttt{species\_id}, con cinco
clases posibles: $\{10,12,17,18,23\}$.

Las variables \texttt{recording\_id}, \texttt{songtype\_id} e \texttt{is\_tp}
fueron tratadas como metadatos y no se incorporaron al espacio de predictores.
La distribuci\'on de clases presenta un desbalance moderado, dominado por las
clases 17 y 23, por lo cual se adopt\'o el \textbf{F1-Score macro} como m\'etrica
principal.

\section{Exploraci\'on Geom\'etrica y Reducci\'on de Dimensionalidad}
\label{sec:dr}
% ===================================================================

Con el objetivo de visualizar la estructura geom\'etrica del espacio
$\mathbb{R}^{64}$ y evaluar la separabilidad de las clases, se compararon dos
enfoques de reducci\'on dimensional: \textbf{PCA}, como m\'etodo lineal, y
\textbf{UMAP}, como m\'etodo no lineal orientado a preservar vecindades locales.

\subsection{Preprocesamiento com\'un}
Se trabaj\'o con 1906 observaciones de entrenamiento y 477 observaciones de prueba.
No se identificaron valores nulos. Las variables predictoras fueron exclusivamente
las 64 columnas Mel. Para evitar fuga de informaci\'on, el escalador
\texttt{StandardScaler} se ajust\'o sobre el conjunto de desarrollo y luego se aplic\'o
sobre validaci\'on y test oficial.

\subsection{PCA y UMAP}
PCA proyecta los datos sobre ejes ortogonales de m\'axima varianza, preservando
estructura global de manera lineal. En la proyecci\'on 2D, PCA retuvo el
$67{,}14\,\%$ de la varianza total, pero las clases se observaron altamente
solapadas. En 3D, la varianza retenida aument\'o a $76{,}77\,\%$.

UMAP, en cambio, prioriza la preservaci\'on de relaciones locales. Su proyecci\'on
2D obtuvo un \textit{trustworthiness} de $0{,}9777$, superior al de PCA 2D
($0{,}8662$), aunque con mayor tiempo de ejecuci\'on. Esto indica que UMAP resulta
m\'as apropiado para visualizaci\'on exploratoria, mientras que PCA conserva valor
como transformaci\'on lineal eficiente e interpretable.

\section{Miner\'ia de Patrones y Estructuras de Clustering}
% ===================================================================

Se evaluaron dos algoritmos de aprendizaje no supervisado: \textbf{Gaussian Mixture
Model (GMM)} y \textbf{DBSCAN}. GMM adopta un enfoque probabil\'istico, modelando los
datos como una mezcla de distribuciones gaussianas, mientras que DBSCAN agrupa
regiones densas y etiqueta como ruido los puntos aislados.

El clustering se aplic\'o sobre representaciones reducidas mediante PCA, con el fin
de disminuir ruido y costo computacional. La selecci\'on de hiperpar\'ametros se
apoy\'o en m\'etricas internas: Silhouette, Davies-Bouldin y Calinski-Harabasz.

\section{Arquitectura de Clasificaci\'on: MLP vs. Modelos de Ensamble}
% ===================================================================

\subsection{Dise\~no de la red neuronal}
Se implement\'o un Perceptr\'on Multicapa (MLP) en PyTorch para clasificaci\'on
multiclase. La funci\'on de p\'erdida utilizada fue la \textbf{entrop\'ia cruzada
categ\'orica}, implementada mediante \texttt{nn.CrossEntropyLoss}. Para una muestra
con logits $\mathbf{z} = (z_1,\dots,z_C)$, la probabilidad predicha para la clase
$k$ se obtiene mediante \textit{softmax}:

\begin{equation}
\hat{p}_k = \frac{e^{z_k}}{\sum_{j=1}^{C} e^{z_j}}, \qquad k = 1,\dots,C.
\end{equation}

La p\'erdida sobre $N$ observaciones se define como:

\begin{equation}
\mathcal{L} = - \frac{1}{N}\sum_{i=1}^{N}\sum_{c=1}^{C}
\mathbb{1}[y_i=c]\log(\hat{p}_{i,c}).
\end{equation}

\subsection{Regularizaci\'on y curvas de aprendizaje}
Se compararon cinco variantes del MLP: modelo base, MLP con Dropout, MLP con
Batch Normalization, Dropout seguido de Batch Normalization y Batch Normalization
seguido de Dropout. La mejor variante fue \textbf{MLP + Dropout post-activaci\'on},
con F1-macro de $0{,}4650$ y accuracy de $0{,}5157$.

\section{Decisiones de Ingenier\'ia y Mitigaci\'on de Riesgos}
% ===================================================================

\subsection{An\'alisis de \textit{trade-offs}}
La selecci\'on del modelo final debe considerar tanto el rendimiento predictivo como
el costo computacional. El MLP + Dropout obtuvo el mejor F1-macro y tambi\'en la
menor latencia de inferencia. XGBoost entren\'o m\'as r\'apido, pero su inferencia fue
m\'as lenta que la del MLP.

\begin{figure}[H]
\centering
\includegraphics[width=0.78\textwidth]{tradeoff_costo_rendimiento.png}
\caption{Compromiso entre costo computacional y rendimiento predictivo.}
\label{fig:tradeoff}
\end{figure}

\subsection{Pol\'itica de umbrales}
Se implement\'o una pol\'itica operativa basada en la probabilidad predictiva
m\'axima $P = \max_k \hat{p}_k$. Se definieron tres zonas:

\begin{itemize}[noitemsep,topsep=2pt]
  \item \textbf{Zona de confianza} ($P \geq 0.85$): clasificaci\'on autom\'atica.
  \item \textbf{Zona de incertidumbre} ($0.40 \leq P < 0.85$): revisi\'on asistida.
  \item \textbf{Zona de rechazo} ($P < 0.40$): descarte autom\'atico.
\end{itemize}

Sobre el conjunto de prueba, la zona de confianza concentr\'o 64 muestras
($13{,}4\,\%$), con accuracy de $0{,}6250$ y F1 de $0{,}4934$. La zona de
incertidumbre incluy\'o 375 muestras ($78{,}6\,\%$), con accuracy de $0{,}4960$ y
F1 de $0{,}4521$. La zona de rechazo agrup\'o 38 muestras ($8{,}0\,\%$), con accuracy
de $0{,}3947$ y F1 de $0{,}3880$.

Esta pol\'itica permite mitigar riesgos operativos: las predicciones con alta
confianza pueden automatizarse, mientras que los casos ambiguos se derivan a
validaci\'on humana.

\begin{figure}[H]
\centering
\includegraphics[width=0.75\textwidth]{politica_umbrales.png}
\caption{Pol\'itica de umbrales basada en la probabilidad m\'axima predicha.}
\label{fig:umbrales}
\end{figure}

\subsection{Despliegue informativo}
Se desarroll\'o una aplicaci\'on en Streamlit para simular inferencia sobre nuevas
observaciones. La aplicaci\'on carga el escalador, el codificador de etiquetas y los
modelos entrenados desde la carpeta \texttt{models/}. El flujo de inferencia aplica
el mismo preprocesamiento usado durante entrenamiento, obtiene probabilidades por
clase y asigna la predicci\'on a una zona operativa seg\'un la pol\'itica de umbrales.

% ===================================================================
\section*{Conclusiones}
% ===================================================================

El proyecto implement\'o un pipeline completo de aprendizaje autom\'atico para
clasificaci\'on eco-ac\'ustica usando 64 caracter\'isticas Mel. La exploraci\'on
geom\'etrica mostr\'o solapamiento entre especies, especialmente en PCA, mientras que
UMAP preserv\'o mejor la estructura local. El clustering no supervisado evidenci\'o
que las especies no forman cinco grupos naturalmente separables en el espacio de
caracter\'isticas.

En clasificaci\'on supervisada, el mejor modelo fue MLP + Dropout, con F1-macro de
$0{,}4650$, seguido por XGBoost con $0{,}4519$. Aunque los resultados reflejan la
dificultad del problema, la pol\'itica de umbrales permite controlar el riesgo de
automatizar predicciones inciertas y aporta una estrategia realista para un sistema
de monitoreo asistido.
